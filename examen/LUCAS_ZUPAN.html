  1. Que es un sistema de control de versiones?

    Los sistemas de control de versiones son un tipo de aplicación con la función de gestionar los cambios de un conjunto de elementos. Deben proporcionar los mecanismos para almacenar los archivos, reliazar cambios sobre estos y registrar historicamente todas las modificaciones.

  2. Cual es la diferencia entre git y github?

    Git es el software de control de versiones, mientras que GitHub es una plataforma donde alojar repositorios de git. Resuelve muchas cosas como gestionar los permisos a otros usuarios para que colaboren en un mismo repositorio, entre otras funciones.

  3. Cómo creo una rama en git?

    Agregar una nueva rama localmente: git branch <nueva_rama>, o con el comando git checkout -b <nueva_rama> para crear y cambiar de rama simultaneamente.
    Agregar una nueva rama al repositorio remoto: git push --set-upstream origin <nueva_rama>

  4. Cómo clono un repositorio?

    Clono un repositorio con el comando: git clone <ruta_repositorio>
    ruta_repositorio: Puede ser una URL para descargar por HTTPS, o un host remoto para descargarlo por SSH (usuario@host:ruta)
